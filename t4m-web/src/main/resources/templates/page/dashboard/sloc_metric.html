Lines of Code

LOC is a metrics to calculate the lines of you project. Currently, T4M only count the number of lines of java source code. T4M provides two statistical methods for counting the LOC, which is based on the source file or based on declared class.

For the methods based on the source file, the java file will be the minimum unit that T4M analysis. Each line will be counted without any formatting.

For the method based on declared class, the minimum unit that T4M analysis is a finer grain than java file. Recall that a java file can declare one public outer class, multiple package-private classes and multiple nested classes. The class will be formatted by JavaParser before being analysed, which split the code statement and the comment into different lines. Besides, the import and package statement will be excluded.

T4M has four measurements for LOC, including the Physical Line of Code (lines that are not empty or not comment-only), the Logical Line of Code (lines that are not empty, not comment-only or not symbol-only) and the Comment Line (lines that contain comments) and the Percentage of Comment Line (the ratio of comment lines to non-empty lines). Empty lines are excluded.

Detailed definitions and influences of LOC/SLOC: Maisqual - Source Lines of Code, ARiSA - Lines of Code, Wiki - Source lines of code.
