Welcome to T4M

T4m is a static code analysis tool for java, designing to be a complement of other similar tools in this domain like SonarQube, Checkstyle, FindBugs, etc. Click here to see the list of other tools.

Click here to see How To Use T4M.

Goals

T4M is aiming to display the trend of software architectural metrics in an interactive and visualized way:

  • To help developers better understand the composition of their project and review the evolution of their project structure during developing.

  • To help developers monitor whether the software is developing as expected, and locating the deviating components.

  • Make it easier to find out possible high-risk packages and classes that are violating the design principles.

Features

T4M currently focus on visualized software metrics (eg. complexity, coupling, cohesion and inheritance metrics) rather than detect the software defects (eg. rule-based detection for code defects, bugs and code smells). When analysis the project, T4M arrange the project on four levels according to the hierarchy, namely module, package, class, method. Different software metrics have different applicable levels.

Remember, the first goal of T4M is showing the changing trends of the metrics.

T4M uses both charts and tables to show these metrics. In each metric page, it comes with a chart showing the overall distribution of the metrics data. You can get a quick view of what your project looks like under these metrics and locate the outliers. We will choose the most suitable chart for different types of metrics to fulfil our first goal. Below the chart is an interactive table, which lists all metrics data hierarchically. For example, you can easily jump into a package to see the metric of every class within. Besides, for every data row, we may also provide a chart to show the changing trends of these metrics.

Supported software metrics

The metrics that T4M supports are selected from Maisqual, ARiSA, including:

Complexity:

  • Lines of Code module package class
  • Percentage of Comments module package class
  • Response for Class class
  • Cyclomatic Complexity method
  • Weighted Methods Count class
  • Average Complexity of Class class
  • Maximum Complexity of Class class

Coupling:

  • Afferent Coupling package class
  • Efferent Coupling package class
  • Instability package class
  • Abstractness package
  • Distance from Main Sequence package
  • Coupling Between Objects class
  • Message Passing Coupling class

Inheritance:

  • Deep of Inheritance Tree class
  • Number of Children class

Cohesion:

  • Lack of Cohesion in Methods class
  • Tight Class Cohesion class
  • Loose Class Cohesion class
